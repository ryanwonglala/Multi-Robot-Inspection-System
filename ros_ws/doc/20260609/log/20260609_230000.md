## 3. sim package (new, TB3/Gazebo Classic)

**File:** `src/sim/worlds/map.world`

Converted the inspection facility SDF from Ignition format (SDF 1.10,
gz-sim plugins) to Gazebo Classic format (SDF 1.6):

- Removed `<plugin filename="gz-sim-*">` blocks
- Replaced with `<plugin name="libgazebo_ros_init.so">` and diff-drive/laser/camera plugins in the robot model
- Added `<light name='sun' type='directional'>` (Gazebo Classic requires explicit sun)
- All 16 floor marker models and facility walls preserved unchanged

**File:** `src/sim/launch/map.launch.py`

Spawns `turtlebot3_burger_cam` via `spawn_entity.py -file model.sdf` (not
`-topic /robot_description`). Key reason: sensor and drive plugins live in
the SDF model, not the URDF. Spawning from the URDF silently skips all
Gazebo plugins and produces a robot with no sensors or movement.

Robot spawn pose: `x=-4.8, y=-3.825, yaw=-1.5708` (charging station).

**File:** `src/sim/scripts/ctl.py`

TB3-specific speed limits: `LIN=0.22 m/s`, `ANG=2.84 rad/s`.

## 5. task_layer package — Nav2 bringup

**File:** `src/task_layer/launch/bringup_nav.launch.py`

Replaces the TB4 bringup which used `turtlebot4_navigation` launch includes.
New version uses `nav2_bringup/launch/bringup_launch.py` directly.
Startup sequence (TimerAction):

- T+0 s: Gazebo + robot spawn (via `sim/launch/map.launch.py`)
- T+8 s: Nav2 full stack (map_server, AMCL, bt_navigator, planners,
  controllers, lifecycle_manager)
- T+8 s: RViz with nav2_default_view config
- T+15 s: `set_initial_pose_node.py` (publishes `/initialpose` 10× at 0.5 s intervals)

**File:** `src/task_layer/config/nav2_inspection.yaml`

Required fixes relative to the TB4 config:

| Issue | Fix |
|---|---|
| Pluginlib `::` separator in plugin class names | Changed to `/` for bt_navigator, planner, smoother, behaviors |
| Costmap layers and controller checkers | Keep `::` (StaticLayer, MPPIController etc. use it in Humble) |
| `enable_stamped_cmd_vel: true` | Set to `false` — Humble uses plain `geometry_msgs/Twist` |
| TB4 footprint radius `0.189 m` | Changed to TB3 Burger radius `0.105 m` |

**File:** `src/task_layer/task_layer/set_initial_pose_node.py`

Publishes `PoseWithCovarianceStamped` to `/initialpose`. Default pose:
`x=0.0, y=0.0, z=0.0325, yaw=0.0`. Repeat-and-exit pattern: 10 publishes
at 0.5 s intervals, then `rclpy.shutdown()`.

## 6. task_layer migration from archive (TB4 → TB3)

All nodes ported from `archive/.../task_layer_v020/` to
`src/task_layer/task_layer/`.

Build type: `ament_cmake` (not `ament_python`). Python modules are
installed as `PROGRAMS` in `CMakeLists.txt`. Required adding
`ament_cmake_python` + `ament_python_install_package(task_layer)` so that
`from task_layer.report_writer import ...` works at runtime.

### File-by-file changes

**scan_analyzer.py** — verbatim copy, zero changes

**report_writer.py** — one line: `default_report_dir()` path changed from
`/home/ryan/tb4_ws/...` to `/home/sutd/roboinspec_ws/reports`

**go_to_area_node.py** — one line: `get_package_share_directory('task_layer_v020')`
→ `'task_layer'`

**inspection_runner.py** — four changes:
- Both internal imports: `task_layer_v020` → `task_layer`
- `get_package_share_directory` package name
- `image_topic` default: `/oakd/rgb/preview/image_raw` → `/camera/image_raw`

**model_spawner.py** — package name rename + major rewrite of spawn command.

TB4 used gz-sim style (Ignition/Harmonic):
```
ros2 run ros_gz_sim create --ros-args -p file:=<f> -p name:=<n> -p x:=<x> ...
```

TB3 uses Gazebo Classic style:
```
ros2 run gazebo_ros spawn_entity.py -file <f> -entity <n> -x <x> -y <y> -z <z> -Y <Y>
```

The SDF temp-file URI rewriting logic (`prepare_sdf_file`) was kept intact.

**scene_builder_gui_node.py** — one line: import path rename

**task_gui_node.py** — four changes:
- Import path rename
- `get_package_share_directory` rename
- `default_report_dir()` path
- Subprocess call: `ros2 run task_layer_v020 inspect_area` → `ros2 run task_layer inspection_runner.py`

Bug fix: added `try/except` guard around `declare_parameter('use_sim_time', True)`.
`use_sim_time` is a built-in ROS2 parameter; redeclaring it raises
`ParameterAlreadyDeclaredException` at node startup.

**task_gui.launch.py** — new file created (did not exist in archive).
Launches `task_gui_node.py` with `world_model_path`, `use_sim_time`,
`world`, `yaw` args. Does not start Gazebo or Nav2 — assumes bringup
is already running.

**models/** — 4 SDF obstacle models copied from archive:
`small_box_obstacle`, `pallet_box_mobile`, `shelf`, pallet jack

## Return-home behavior note

`inspection_runner.py` computes the return pose from `world_model.yaml →
robot_start → pose` with a configurable standoff:

```
return_x = pose.x - cos(yaw) × standoff_distance
return_y = pose.y - sin(yaw) × standoff_distance
```

Default: `return_home_standoff_distance = 0.6 m`. With `yaw=-1.5708`
(south-facing dock), the robot stops 0.6 m north of the charging marker.
Disable with `return_home:=false` or override target with `home_area:=<key>`.
