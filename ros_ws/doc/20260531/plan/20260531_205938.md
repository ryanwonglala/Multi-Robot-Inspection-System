# Plan 20260531 Semantic World Model

## Goal

Turn the current inspection facility SDF into a task-layer semantic world model.

## Reason

Task commands should target meaningful areas such as `lab_room`, `server_room`, or `narrow_passage`, not raw coordinates. The world model translates semantic areas into concrete navigation and inspection poses.

## Scope

- Compare `demo.sdf` and `map.sdf`.
- Use `map.sdf` as the current simulation source.
- Extract semantic areas from floor markers and facility layout.
- Add `world_model.yaml` to the dry-run experiment.
- Update `task_runner.py` so `InspectArea(area)` expands into entry poses, inspect poses, scan profiles, image capture, and return home.

## Next Step

Review the generated area names and pose coordinates in Gazebo/RViz, then adjust unsafe or awkward points before connecting `NavigateTo` to Nav2.
