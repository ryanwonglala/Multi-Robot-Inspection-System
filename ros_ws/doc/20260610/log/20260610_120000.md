## 3. Known remaining limitation

**Nav timeout absent in `inspection_runner.py`**

`send_goal_and_wait()` has no timeout on the navigation goal itself.
`server_timeout_sec` (default 10 s) only guards action-server availability,
not goal execution time. When an obstacle occupies the goal cell, Nav2 loops
on replanning indefinitely and the runner never proceeds to fallback candidates.

The fallback candidate mechanism (`generate_candidate_poses()` +
`max_candidate_attempts_per_area`) is structurally correct and will work
once a `nav_timeout_sec` parameter + cancel-on-timeout is added to
`send_goal_and_wait()`. Deferred to next development phase.
