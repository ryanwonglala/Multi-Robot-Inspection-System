# Dbg 20260527_005455

## Symptom

The local `ctl.py` GUI opened but did not move TurtleBot4.

Manual velocity command that worked:

```bash
ros2 topic pub --rate 10 /cmd_vel geometry_msgs/msg/TwistStamped \
"{header: {stamp: {sec: 0, nanosec: 0}, frame_id: ''}, twist: {linear: {x: 0.15, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}}"
```

## Root Cause

The GUI published `geometry_msgs/msg/Twist` to `/cmd_vel`.

Official Create3 Gazebo bridge expects `/cmd_vel` as:

```text
geometry_msgs/msg/TwistStamped
```

Confirmed in:

```text
/opt/ros/jazzy/share/irobot_create_gz_bringup/launch/create3_ros_gz_bridge.launch.py
```

Relevant bridge mapping:

```text
/cmd_vel@geometry_msgs/msg/TwistStamped[ignition.msgs.Twist
```

## Fix

Changed `src/sim/scripts/ctl.py` to publish `TwistStamped` with a header timestamp and nested `twist` field.
