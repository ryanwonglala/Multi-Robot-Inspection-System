## Issue 1: Spawn Direction

TurtleBot4 spawns together with the official charging dock. This is expected because official `turtlebot4_spawn.launch.py` spawns both:

- `turtlebot4`
- `standard_dock`

The dock pose is calculated from the robot yaw with a fixed offset. The default local yaw has been rotated by 180 degrees for the current map entrance pose.

## Issue 2: GUI Keyboard Control

The Gazebo GUI config contains a `/cmd_vel` teleop topic, but manual testing showed GUI keyboard control does not move the robot.

Direct ROS velocity commands do move the robot, so the control stack itself is working.

Current interpretation:

- Treat Gazebo GUI teleop as not reliable for this project path.
- Use ROS-side teleop or direct `/cmd_vel` commands for mapping.
- Later, if needed, create a local minimal teleop tool or inspect Gazebo GUI teleop bridge wiring.
