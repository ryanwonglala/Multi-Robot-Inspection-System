# Dbg 20260527_012038

## Current SDF Editability Problem

The wall geometry is currently stored as many `<link>` elements inside one model:

```xml
<model name="small_indoor_technical_facility_walls">
  <static>true</static>
  <link name="outer_north">...</link>
  <link name="outer_west">...</link>
  ...
</model>
```

Gazebo GUI commonly treats the model as the main selectable/editable entity. Because all wall links are grouped inside one model, selecting/copying/moving one wall segment in GUI is difficult or impossible.

## Recommended Format Change

Convert each wall segment from a link inside one model into its own static model:

```xml
<model name="wall_outer_north">
  <static>true</static>
  <pose>0 6.5 0.6 0 0 0</pose>
  <link name="link">
    <collision name="collision">...</collision>
    <visual name="visual">...</visual>
  </link>
</model>
```

This gives each wall segment its own model name in Gazebo and should make GUI selection, copy, paste, and movement more practical.

## Migration Rule

For each current wall link:

```xml
<link name="X"><pose>P</pose><collision>...</collision><visual>...</visual></link>
```

Create:

```xml
<model name="wall_X">
  <static>true</static>
  <pose>P</pose>
  <link name="link">
    <collision>...</collision>
    <visual>...</visual>
  </link>
</model>
```

Remove the pose from the link and put it on the model.

## Caution

This is a structural migration of the world file. Do it in one pass with XML parsing, then validate by launching Gazebo and checking that all walls appear in the same locations.
