# Dbg 20260527_011440

## Problem

Manual SLAM drift appeared when TurtleBot4 reached the upper-right region around the blue/orange/purple zones.

## Hypothesis

The narrow outer-wall passage near the right-side rooms created ambiguous scan matching:

- long parallel wall segments
- repeated short wall corners
- limited distinctive LiDAR features
- tight navigation space near the outer wall

This made local scan alignment vulnerable to drift.

## Fix Applied

Added a clean internal east boundary at approximately:

```text
x = 8.75
 y = -6.0 to 6.5
```

The new wall prevents the robot from entering the outer-wall strip while preserving the main facility layout.

Two short horizontal anchor walls were added to break up the long wall and create more distinctive LiDAR features:

```text
x = 8.2, y = 2.55
x = 8.2, y = -1.0
```
