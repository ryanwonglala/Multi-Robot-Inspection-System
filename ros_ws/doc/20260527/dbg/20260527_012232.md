# Dbg 20260527_012232

## Why This Migration Was Needed

Gazebo GUI appeared unable to select individual wall segments because they were grouped as links inside one large static model.

The old structure was optimized for compact SDF text, not manual GUI editing.

## Migration Rule Used

For each old wall link:

```xml
<link name="X"><pose>P</pose><collision>...</collision><visual>...</visual></link>
```

Created:

```xml
<model name="wall_X">
  <static>true</static>
  <pose>P</pose>
  <link name="link"><collision>...</collision><visual>...</visual></link>
</model>
```

The pose moved from link level to model level.

## Geometry Promise

No wall dimensions or poses were intentionally changed.


## Manual Validation Needed

1. Launch Gazebo.
2. Open entity tree or selection tool.
3. Confirm names such as `wall_outer_north`, `wall_server_west`, `wall_restricted_west` appear as separate models.
4. Try selecting/copying/moving one wall segment.

## Caveat

The walls remain `<static>true</static>`. If Gazebo GUI still refuses live movement of static models, use one of these paths later:

- edit pose values directly in SDF
- temporarily set selected wall models to static false while editing
- use Gazebo's model editing workflow if available
