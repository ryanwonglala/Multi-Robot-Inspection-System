# Commands

## Build after map changes

```bash
cd /home/ryan/tb4_ws
source /opt/ros/jazzy/setup.bash
colcon build
source install/setup.bash
```

## Launch Gazebo/TurtleBot4 without SLAM

```bash
ros2 launch sim map.launch.py slam:=false rviz:=true x:=-3.575 y:=-3.3 yaw:=3.1416
```

## Launch localization with saved map

```bash
ros2 launch turtlebot4_navigation localization.launch.py \
  use_sim_time:=true \
  map:=/home/ryan/tb4_ws/maps/base_20260527.yaml
```

## Launch Nav2

```bash
ros2 launch turtlebot4_navigation nav2.launch.py use_sim_time:=true
```

## Suggested RViz validation goals

```text
mother_base:   x=-3.575, y=-3.3
main_corridor: x=0.0,    y=-2.6
central_hall:  x=0.0,    y=1.2
lab_room:      x=-4.25,  y=4.25
server_room:   x=4.5,    y=4.85
```

Use `2D Pose Estimate` carefully. The robot heading must match the actual robot direction; a reversed initial heading can cause planning/navigation failures.
