# Next plan

## Recommended next milestone

Build the first real ROS2 task-layer node that connects semantic area names to Nav2 goals.

Target demo:

```text
structured area task -> lookup area in world_model.yaml -> send NavigateToPose -> report status
```

Example task:

```yaml
task:
  type: go_to_area
  target: lab_room
```

Expected behavior:

```text
Read world_model.yaml
Find lab_room.center = [-4.25, 4.25]
Send Nav2 NavigateToPose goal
Wait for result
Print succeeded/failed
```

## Suggested implementation order

1. Create a dedicated ROS2 package, likely `src/task_layer`.
2. Add a Python node, e.g. `task_runner_node.py`.
3. Copy or install the semantic `world_model.yaml` as package config.
4. Accept a target area parameter first, e.g. `target:=lab_room`.
5. Send a `nav2_msgs/action/NavigateToPose` action goal.
6. Test from `mother_base` to these areas:
   - `main_corridor`
   - `central_hall`
   - `lab_room`
   - `server_room`

## Notes

Do not start with multi-robot yet. First prove the single-robot task-to-Nav2 loop works. Once stable, extend to multi-target task decomposition and robot assignment.
